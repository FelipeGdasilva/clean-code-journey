# 🚀 Minha Jornada Clean Code

> *"Código limpo sempre parece que foi escrito por alguém que se importa."* — Michael Feathers

Este repositório documenta meu aprendizado durante a leitura do livro **Código Limpo (Robert C. Martin)**. Aqui, aplico os conceitos teóricos na minha stack (Node.js, React e JavaScript) para transformar código funcional em código profissional.

---

## 🛠️ Princípios que estou dominando

* **Nomes Significativos:** Variáveis e funções que revelam sua intenção sem precisar de comentários.
* **Funções Pequenas:** Funções que fazem apenas uma coisa e de forma excelente.
* **Regra do Escoteiro:** "Deixe o código sempre um pouco mais limpo do que você o encontrou".
* **Tratamento de Erros:** Antecipar falhas de forma limpa, como aprendi ao integrar IA em automações com n8n.

---

## 📂 Organização dos Estudos

Cada pasta contém exemplos de **código sujo (dirty.js)** comparado com sua **versão refatorada (clean.js)**:

1. `01-nomes-significativos`: Exemplos de como nomear constantes e funções.
2. `02-funcoes-pequenas`: Técnicas de refatoração para diminuir a complexidade.
3. `03-comentarios-desnecessarios`: Como escrever código que se explica sozinho.
4. `04-tratamento-de-erros`: Estratégias para um código mais resiliente.

---

## 📈 Evolução Prática

Aplico estes conceitos em meus projetos principais para consolidar o conhecimento:

* **Desafios Front-end:** Refatoração de componentes (ex: Interactive Rating Component) aplicando Clean Code.
* **Projetos Back-end:** Organização de rotas e lógica de negócio com foco em legibilidade.
* **Automações:** Otimização de prompts e fluxos no n8n para maior flexibilidade.

Meu objetivo é transicionar para a área de tecnologia, entregando software de alta qualidade, legível e de fácil manutenção.
