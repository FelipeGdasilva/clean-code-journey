# 🚀 Minha Jornada Clean Code

> "Código limpo sempre parece que foi escrito por alguém que se importa." — Michael Feathers

Este repositório documenta meu aprendizado durante a leitura do livro **Código Limpo (Robert C. Martin)**. Aqui, aplico os conceitos teóricos na minha stack de estudo (Node.js, React e JavaScript) para transformar código funcional em código profissional.

---

## 🛠️ Princípios que estou dominando

* **Nomes Significativos:** Variáveis e funções que revelam sua intenção sem precisar de comentários.
* **Funções Pequenas:** Funções que fazem apenas uma coisa e de forma excelente.
* **Regra do Escoteiro:** "Deixe o código sempre um pouco mais limpo do que você o encontrou".
* **Tratamento de Erros:** Antecipar falhas de forma limpa, como aprendi ao integrar IA nas minhas automações com n8n.

---

## 📂 Organização dos Estudos

O repositório está dividido por capítulos para facilitar a consulta:

1.  `01-nomes-significativos`: Exemplos de como nomear constantes e funções.
2.  `02-funcoes-pequenas`: Técnicas de refatoração para diminuir a complexidade.
3.  `03-comentarios-desnecessarios`: Como escrever código que se explica sozinho.
4.  `04-tratamento-de-erros`: Estratégias para um código mais resiliente.

---

## 📈 Evolução Prática

Aplico esses conceitos nos meus projetos principais para consolidar o conhecimento:
* **Desafios Front-end:** Refatoração do componente de rating (usando boas práticas de nomes como `submitButton`).
* **Projetos Back-end:** Organização de rotas e lógica de negócios focada em legibilidade.
* **Automações:** Melhoria na estrutura de prompts e fluxos no n8n.
